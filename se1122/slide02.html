What is Computation? & The Basic Elements of Python

Chapter 1: What is Computation?

Computers can perform trillions of calculations per second, yet they possess no innate intelligence. Understanding computation means understanding the nature of instructions.

  • What types of knowledge can we encode?
  • How do computational processes execute those types of knowledge?
  • How did we transition from dedicated machines to general-purpose computing?

Declarative vs. Imperative

The core dichotomy of knowledge engineering:

  • Declarative Knowledge: Statements of fact. It defines truth or relationships but offers no guidance on how to reach them.
    Example: "The square root of a number $x$ is a number $y$ such that $y \cdot y = x$."
  • Imperative Knowledge: A description of a method; a "recipe" consisting of a sequence of step-by-step instructions.
    Example: "Start with an initial guess. Square it. If the difference is close enough, stop."

Example 1: Testing Truth (Declarative Approach)

A declarative definition can check if a solution satisfies a condition, but it doesn't calculate the answer from scratch.

Try running this at: https://www.programiz.com/python-programming/online-compiler/

x = 16
possible_root = 4

# Checking a declarative condition (y * y == x)
is_square_root = (possible_root * possible_root == x)

print("Is 4 the square root of 16?")
print(is_square_root) # Evaluates to True

Analysis Steps:

  • 1. The state of variables x and possible_root are stored.
  • 2. The comparison 4 * 4 == 16 evaluates directly to a boolean literal.
  • 3. This pattern only verifies state; it does not generate a missing value when possible_root is unknown.

The Computational Recipe (Algorithm)

An effective recipe or algorithm must contain three crucial properties:

  • Sequence of steps: A clear, deterministic ordering of actions.
  • Flow of control: The capacity to branch or repeat actions based on conditions.
  • Termination condition: A clear state where execution finishes with a definitive result.

Example 2: Heron's Method Imperative Algorithm

An ancient imperative recipe (Heron's Method) to compute successive improvements for an approximate square root:

x = 16.0
guess = 1.0 # Initial absolute guess
epsilon = 0.001
iterations = 0

while abs(guess**2 - x) >= epsilon:
    # Refine the guess using the average: (guess + x/guess) / 2
    guess = (guess + (x / guess)) / 2.0
    iterations += 1

print(f"Result: {guess} after {iterations} steps")

Trace Steps Tracking:

  1. guess = 1.0: abs(1.0 - 16.0) = 15.0 >= 0.001. Loop enters. guess = (1.0 + 16.0)/2 = 8.5.
  2. guess = 8.5: abs(72.25 - 16.0) = 56.25. Loop continues. guess = (8.5 + 16/8.5)/2 = 5.191.
  3. guess = 5.191: Next iteration calculates guess = 4.136.
  4. guess = 4.136: Next iteration reaches 4.00000037 which meets the validation limit, and terminates safely.

Evolution of Computing Architecture

  • Fixed-Program Computers: Early machines where circuitry dictated a single calculation profile (e.g., calculations of mathematical tables or ballistic sheets). Altering the task meant physically rebuilding or rewiring the machine.
  • Stored-Program Computers: Alan Turing and John von Neumann shifted this paradigm by proposing that instructions should be stored as data configurations in the computer's memory.
  • The computer uses an interpreter to fetch, decode, and execute instructions sequentially from memory.

Subchapter 2.1: The Elements of Python

Programming languages have structural characteristics analogous to natural languages:

  • Primitives: The basic building blocks (characters, symbols, literal data types like integers or strings).
  • Syntax: Rules determining whether a structural configuration of primitives is well-formed (e.g., 3 + 5 is syntactically correct; 3 + is a syntax error).
  • Static Semantics: Rules that validate whether a syntactically valid statement makes sense before execution (e.g., 'abc' + 3 has correct syntax but causes a static semantic error because you cannot combine strings and integers).
  • Semantics: The actual meaning or behavior of the code when run. A program can run without errors but still contain semantic bugs if it calculates the wrong result.

Example 3: Categorizing Code Faults

Let's dissect three distinct programming errors according to Python's structural rules:

# 1. Syntax Error (Malformed structure)
# total = 5 + * 2 

# 2. Type/Static Semantic Error (Incompatible types)
# length = "10" + 5

# 3. Full Semantic Bug (Valid code, incorrect logic)
base = 10
height = 5
area = base * height # Forgot to divide by 2 for a triangle!

Impact Analysis:

  • Line 2 prevents execution immediately because Python cannot build an abstract syntax tree.
  • Line 5 causes a runtime TypeError when evaluated because concatenation/addition requires matching types.
  • Line 10 executes perfectly from Python's perspective, but outputs an incorrect mathematical result (50 instead of 25).

Objects and Scalar Types

In Python, everything is an object. Objects occupy a space in memory, have an explicit type, and can be categorized into two groups:

  • Scalar objects: Atomic, fundamental values that cannot be broken down further.
  • Non-scalar objects: Objects with internal structural components (e.g., lists or dictionaries).

The 4 Core Scalar Types in Python:

  • int: Represents integers (e.g., -5, 42).
  • float: Represents real floating-point numbers (e.g., 3.14159, 2.0).
  • bool: Represents Boolean values (True or False).
  • NoneType: Has exactly one value (None), indicating the absence of a value.

Subchapter 2.2: Operators and Overloading

Operators combine scalar types to form computational expressions. Their behavior changes based on the data types involved (known as operator overloading).

  • i + j: Addition. If both are ints, returns an int. If either is a float, returns a float.
  • i / j: Floating-point division. Always returns a float (e.g., 6 / 2 yields 3.0).
  • i // j: Integer division. Truncates fractional remainders, rounding down to the next integer.
  • i ** j: Exponentiation ($i^j$).
  • i % j: Modulo operator. Returns the integer remainder of division.

Example 4: Tracing Arithmetic Operator Precedence

Let's trace how Python evaluates mixed scalar types step-by-step according to operator precedence:

expr1 = 11 // 3 + 2.5 * 2
expr2 = 11 % 3 ** 2
expr3 = (11 % 3) ** 2

Step-by-Step Evaluation:

  • expr1: 11 // 3 evaluates to 3. 2.5 * 2 evaluates to 5.0. Then 3 + 5.0 returns 8.0 (float).
  • expr2: Exponentiation happens first. 3 ** 2 is 9. Then 11 % 9 evaluates to 2.
  • expr3: Parentheses override standard precedence. 11 % 3 is 2. Then 2 ** 2 evaluates to 4.

Variables and Binding Assignments

Variables act as descriptive labels that point to explicit memory locations where objects reside.

  • An assignment statement using the = operator binds a name reference to an object target.
  • Python variables do not have inherent, unchangeable types. Instead, the objects they point to have types (this is called dynamic typing).
  • Reassigning a variable updates its reference link to point to a new object, leaving the old object behind.

Example 5: Memory Reference Binding Chain

Let's follow how assignments change variable references in memory step-by-step:

x = 100
y = x

x = x + 50
y = y / 2

final_result = x - y

Trace Steps Tracking:

  1. x references an int object containing 100. y = x binds y to point to that same exact int object.
  2. x = x + 50: Evaluates 100 + 50 = 150. A new int object 150 is created, and x is updated to point to it. Crucial: y still points to the original object 100.
  3. y = y / 2: Evaluates 100 / 2 = 50.0. y updates to point to this new float object.
  4. final_result: Computes 150 - 50.0, resulting in the float value 100.0.

Summary: Foundations of Python

  1. Computation: Moves from fixed tasks to flexible, stored-program instruction execution handled by interpreters.
  2. Language Safeguards: Python filters out structural errors using syntax and static semantic checking before code semantics run.
  3. Objects & Typing: Data primitives are stored as typed scalar objects. Variables act as independent, dynamic reference labels pointing to those objects.
