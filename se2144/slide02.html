Practical Application

Building Your First
NextJS App

A comprehensive, step-by-step guide applying the core concepts from Chapter 1.

Focus: Environment, Routing, Components, and the Pages Directory.

What Are We Building?

We will build a simple Developer Portfolio & Blog. It will demonstrate:

  • Project Initialization: Setting up with TypeScript.
  • Component Architecture: Building a reusable Layout.
  • Static Routing: Creating Home and About pages.
  • Navigation: Client-side transitions using next/link.
  • Dynamic Routing: Creating a dynamic blog post reader.
  • Styling: Applying CSS Modules.
Step 1

Environment & Init

First, open your terminal. Navigate to your desired workspace folder and run the create-next-app command. We will use the traditional Pages router for this exercise.


# Initialize a new Next.js project named "my-portfolio"
yarn create next-app my-portfolio

Tailwind Configuration

Because we selected "Yes", Next.js automatically generated tailwind.config.js and postcss.config.js. It also pre-populated our global CSS.

Actions:
  1. Go to pages/index.tsx and delete everything inside.
  2. Open styles/globals.css. Keep the top Tailwind directive, but delete the rest of the boilerplate CSS below it.
  3. Delete the pages/api folder (we don't need backend routes yet).

Step 3

The Global Wrapper

Let's look at pages/_app.tsx. This is the root component that wraps every page in your application. It's where you import global CSS.


// pages/_app.tsx
import '@/styles/globals.css'
import type { AppProps } from 'next/app'

export default function App({ Component, pageProps }: AppProps) {
  // 'Component' is the active page (e.g., index.tsx, about.tsx)
  return <Component {...pageProps} />
}
Concept Check: You cannot import global CSS anywhere except here. Component-specific CSS (Modules) will be used everywhere else.
Step 4

Building a Reusable Layout

Let's create a standard navigation bar that appears on every page. Create a new folder named components/ in the root, and add Layout.tsx.


// components/Layout.tsx
import Link from 'next/link';
import { ReactNode } from 'react';

interface LayoutProps {
  children: ReactNode;
}

export default function Layout({ children }: LayoutProps) {
  return (
    <div className="min-h-screen bg-slate-900 text-slate-100 font-sans">
      <nav className="p-6 bg-slate-950 border-b border-slate-800 flex gap-6">
        <Link href="/" className="hover:text-sky-400 transition-colors font-semibold">Home</Link>
        <Link href="/about" className="hover:text-sky-400 transition-colors font-semibold">About</Link>
      </nav>
      {/* The actual page content is injected here */}
      <main className="max-w-4xl mx-auto p-8">{children}</main>
    </div>
  );
}
Step 5
Applying the Layout & next/link

Notice the use of <Link> in the previous slide? This is NextJS's built-in client-side router component.

  • Instead of <a href="/about">, we use <Link href="/about">.
  • Why? It prevents the browser from doing a full page refresh. It fetches the JavaScript for the new page instantly.

To apply our Layout globally, let's update _app.tsx:


// pages/_app.tsx
import '@/styles/globals.css'
import Layout from '@/components/Layout'
import type { AppProps } from 'next/app'

export default function App({ Component, pageProps }: AppProps) {
  return (
    <Layout>
      <Component {...pageProps} />
    </Layout>
  )
}

Step 8

Dynamic Routing Setup

What if we have 100 blog posts? We don't want to create 100 files. We use Dynamic Routes by using brackets in the file name: [id].tsx.

File Structure:

Create a folder named post inside pages. Inside it, create [id].tsx.


pages/
├── index.tsx
├── about.tsx
└── post/
    └── [id].tsx  <-- The dynamic segment
Any URL like /post/hello-world or /post/123 will automatically be routed to this [id].tsx file!
Step 9

Reading Dynamic Parameters

Inside [id].tsx, we use the useRouter hook to read the URL parameter.


// pages/post/[id].tsx
import { useRouter } from 'next/router';

export default function BlogPost() {
  const router = useRouter();
  const { id } = router.query; 

  return (
    <div className="mt-10 p-8 rounded-2xl bg-gradient-to-b from-slate-800 to-slate-900 border border-slate-700 shadow-xl">
      <span className="text-xs font-bold tracking-widest uppercase text-slate-400">Current Path ID:</span>
      <h1 className="text-5xl font-black text-pink-500 capitalize mt-2 mb-6">{id}</h1>
      <p className="text-slate-300">This page was generated dynamically based on the URL parameter.</p>
    </div>
  )
}
