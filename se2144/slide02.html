Building Your First
NextJS App

A comprehensive, step-by-step guide applying the core concepts from Chapter 1.

Focus: Environment, Routing, Components, and the Pages Directory.

What Are We Building?

We will build a simple Developer Portfolio & Blog. It will demonstrate:

  • Project Initialization: Setting up with TypeScript.
  • Component Architecture: Building a reusable Layout.
  • Static Routing: Creating Home and About pages.
  • Navigation: Client-side transitions using next/link.
  • Dynamic Routing: Creating a dynamic blog post reader.
  • Styling: Applying CSS Modules.
Step 1

Environment & Init

First, open your terminal. Navigate to your desired workspace folder and run the create-next-app command. We will use the traditional Pages router for this exercise.

Step 3

The Global Wrapper

Let's look at pages/_app.tsx. This is the root component that wraps every page in your application. It's where you import global CSS.


// pages/_app.tsx
import '@/styles/globals.css'
import type { AppProps } from 'next/app'

export default function App({ Component, pageProps }: AppProps) {
  // 'Component' is the active page (e.g., index.tsx, about.tsx)
  return 
}
Concept Check: You cannot import global CSS anywhere except here. Component-specific CSS (Modules) will be used everywhere else.

Step 5

Applying the Layout & next/link

Notice the use of <Link> in the previous slide? This is NextJS's built-in client-side router component.

  • Instead of <a href="/about">, we use <Link href="/about">.
  • Why? It prevents the browser from doing a full page refresh. It fetches the JavaScript for the new page instantly.

To apply our Layout globally, let's update _app.tsx:


import '@/styles/globals.css'
import Layout from '@/components/Layout'
import type { AppProps } from 'next/app'

export default function App({ Component, pageProps }: AppProps) {
  return (
    
      
    
  )
}

Step 8

Dynamic Routing Setup

What if we have 100 blog posts? We don't want to create 100 files. We use Dynamic Routes by using brackets in the file name: [id].tsx.

File Structure:

Create a folder named post inside pages. Inside it, create [id].tsx.


pages/
├── index.tsx
├── about.tsx
└── post/
    └── [id].tsx  <-- The dynamic segment
Any URL like /post/hello-world or /post/123 will automatically be routed to this [id].tsx file!
Step 9

  const router = useRouter();
  const { id } = router.query; // 'id' matches the filename [id].tsx
